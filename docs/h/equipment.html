Биллинговая система NoDeny. Модуль оборудования

NoDeny позволяет вести учет оборудования с привязкой к владельцам/месторасположению. В качестве оборудования может выступать любой объект, например, свич, сетевая карта, инструмент и т.д.

Каждый тип оборудования имеет свои персональные характеристики, скажем, у свича есть такие параметры как серийный номер, модель, количество портов, стомость, поставщик, срок гарантии, логин, пароль, ip-адрес и т.д. У сетевой карты: мак-адрес, серийный номер, поставщик. Таким образом, необходимо сообщить NoDeny о структуре каждого типа оборудования. Для этого заходим в раздел «Операции» → «Настройки» → «Типы оборудования» и вносим несколько основных типов, например «Свичи» и «Сетевые карты», нажимаем «Сохранить».

В разделе настроек переходим в подраздел «Дополнительные поля» и нажимаем «Создать новое». В появившемся окне создания дополнительного поля выбираем шаблон «Свичи», в параметр «Название поля» впечатываем «Серийный номер», «Алиас» оставляем пустым, «Тип поля» - в значение «целое». нажимаем «Сохранить». Только что мы создали один параметр для типа оборудования «Свичи». Создайте остальные, которые считаете нужными (производитель, модель, количество портов и т.д.).

Среди параметров есть галочка «титульное поле». Эта галочка влияет на поиск. Допустим вы хотите найти какую-либо единицу (или список по заданным критериям) оборудования. Поскольку у оборудования может быть большое количество полей, то вывод всех полей в результатах поиска приведет к ухудшению восприятия информации - слишком много лишних данных. Поэтому в результах поиска будут выводится только те поля, у которых установлена галочка «титульное поле». Например, поставьте ее у полей «Серийный номер» и «Название».

Среди типов полей есть «привязка к объекту» - указывает за кем в данный момент закреплена конкретная единица оборудования. При создании (внесении в базу) единицы оборудования, последняя привязывается к администратору, создавшего ее. Т.е. он является ее владельцем или, другими словами, созданная единица оборудования числится за администратором.

Таким администратором может быть кладовщик, который при приеме оборудования вносит его в базу и это оборудование закрепляется за ним. Далее каждая единица оборудования может сменить владельца, например, передана работнику, потом установлена в коммуникационный ящик (привязана к точке топологии). Либо же, в случае с сетевой картой, последняя может быть передана клиенту.

Поле «привязка к объекту» не является обязательным, однако оно позволит в любой момент времени ответить на вопрос «где сейчас находится свич такой-то?», при этом отследить всю историю его перемещений.
